# RTMify Trace — Validation Package Sources

This directory contains the committed source materials for the RTMify Trace IQ/OQ Validation Package.

The shipped validation package is generated by [`/Users/colinsteele/Projects/rtmify/sys/release.sh`](/Users/colinsteele/Projects/rtmify/sys/release.sh) and emitted under:

- `sys/dist/<version>/validation/package/`
- `sys/dist/<version>/validation/RTMify_Trace_Validation_Package_v<version>.zip`

## Committed sources

| File | Description |
|------|-------------|
| `prd.md` | Specification for package contents, expected results, and acceptance criteria |
| `expected-gaps.json` | Machine-readable semantic oracle for the profile-enabled OQ run |
| `gen_oq_fixture.py` | Legacy generic fixture generator retained for reference |
| `RTMify_OQ_Fixture_v0.1.0.xlsx` | Committed 7-tab profile-enabled fixture source reference |
| `protocol.html` | Editable protocol source template |
| `evidence.html` | Editable blank evidence-record source template |
| `print.css` | Shared print stylesheet for generated PDFs |
| `render_pdf.mjs` | HTML-to-PDF renderer using Playwright/Chromium |
| `package_validation.py` | Validation package generator and semantic verifier |
| `test_package_validation.py` | Validation packaging regression tests |

## Generated release artifacts

The validation package generator produces a package bound to the `medical` profile:

- `README.txt`
- `RTMify_OQ_Fixture_v<version>.xlsx`
- `golden/golden.pdf`
- `golden/golden.docx`
- `golden/golden.md`
- `golden/golden-gaps.json`
- `RTMify_Trace_IQOQ_Protocol_v<version>.pdf`
- `RTMify_Trace_IQOQ_Evidence_v<version>.pdf`
- `checksums.txt`

Those files are release artifacts. They are not maintained by hand in this directory.

## Local regeneration

Fixture only:

```sh
cd /Users/colinsteele/Projects/rtmify/sys/validation
python3 gen_oq_fixture.py --version 0.1.0
```

Validation package only:

```sh
python3 /Users/colinsteele/Projects/rtmify/sys/validation/package_validation.py \
  --version 0.1.0 \
  --trace-binary /abs/path/to/rtmify-trace \
  --trace-binary-windows /abs/path/to/rtmify-trace.exe \
  --trace-binary-linux /abs/path/to/rtmify-trace-linux \
  --out-dir /abs/path/to/dist/0.1.0/validation/package \
  --checksums-file /abs/path/to/dist/0.1.0/validation/package/checksums.txt
```

## Release flow

The blessed release path is:

```sh
cd /Users/colinsteele/Projects/rtmify/sys
./release.sh
```

That flow:

1. builds Trace/Live release artifacts
2. builds Linux/Windows/macOS Trace binaries for checksum publication
3. regenerates the validation package under `--profile medical`
4. verifies `golden-gaps.json` semantically against `expected-gaps.json`
5. verifies the same fixture under `--profile generic` does not emit profile E12xx gaps
6. emits the final ZIP into `dist/<version>/validation/`
